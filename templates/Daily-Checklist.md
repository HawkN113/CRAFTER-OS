# Daily Checklist (Daily Playbook)

## 🟦 PHASE 1: Cognitive Clarity
*Establish focus before execution.*

- [ ] **Define Top 1–3 Tasks:** Identify high-leverage outcomes.
    * *Example: "Implement JWT validation middleware" instead of "Work on Auth"*
- [ ] **Clarity Check:** Do I have clear requirements and a "Definition of Done"?
- [ ] **Time Block:** Allocate 2–4 hours for uninterrupted Deep Work.

---

## 🟩 PHASE 2: Execution
*Operating the engineering engine.*

- [ ] **Noise Reduction:** Disable notifications (Slack, Email, Phone).
- [ ] **Single-Tasking:** Work strictly on Task #1. No context switching.
- [ ] **Stall Protocol:** If stuck for >20 mins, stop and re-map the problem on paper/whiteboard.

---

## 🟨 PHASE 3: Result & Delivery
*Closing the loop and securing reputation.*

- [ ] **Quality Audit:** Does the code meet system standards? (Tests, Clean Code, Docs)
- [ ] **Result Delivered:** PR submitted, stakeholder notified, or task moved to 'Done'.
- [ ] **Reputation Check:** Did I deliver what I promised today?

---

## 🟥 PHASE 4: Reflection
*Data for the Continuous Feedback Loop.*

- [ ] **Predictability Score:** ___% (Tasks completed / Tasks planned)
- [ ] **Deep Work Hours:** ___h
- [ ] **Key Blocker:** What was the main distraction or bottleneck today?
- [ ] **Tomorrow's Seed:** What is the first 5-minute action for tomorrow?

---

### 💡 Example Entry
- **Task:** [PR] Refactor Database Connector for Connection Pooling.
- **Clarity:** 🟢 Requirements confirmed by Lead. Specs in Jira-402.
- **Outcome:** PR #102 opened. 100% test coverage.
- **Reflection:** Spent too much time on a bug that was out of scope. Tomorrow: Stick to the task boundary.
