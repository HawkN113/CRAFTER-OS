# 🤝 Contributing to CRAFTER OS

CRAFTER is a system, not a random collection of ideas.

---

## Principles

Contributions must:

- align with CRAFTER principles
- improve clarity, not complexity
- be actionable, not theoretical
- be minimal and structured

---

## What is accepted

- improvements to clarity
- better execution practices
- measurable metrics
- real-world use cases

---

## What is NOT accepted

- vague ideas
- over-engineering
- unnecessary abstractions

---

## Process

1. Fork repository
2. Create branch
3. Submit PR with clear explanation
4. Link to a problem being solved

---

## Rule

> If it does not improve execution — it does not belong here.
