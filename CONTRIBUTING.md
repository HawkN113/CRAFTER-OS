# 🤝 Contributing to CRAFTER OS

CRAFTER is a **deterministic system**, not a collection of random ideas. We value contributions that sharpen the tools, clarify the mindset, and improve the execution of engineers worldwide.

---

## 💎 The Contribution Filter

Every Pull Request must pass the **CRAFTER Test**:
1. **Does it align with Core Principles?** (See [CRAFTER.md](./CRAFTER.md)).
2. **Does it improve Clarity?** (Does it make the system easier to understand?).
3. **Is it Actionable?** (Can an engineer apply this tomorrow morning?).
4. **Is it Minimal?** (Did you remove the "noise" and leave only the "signal"?).

---

## ✅ What We Welcome

We are looking for "System Patches" in the following areas:
- **Clarity Upgrades:** Refinement of definitions, better metaphors, or improved documentation structure.
- **Execution Protocols:** New checklists or rituals that have been proven to increase velocity.
- **Measurable Metrics:** New ways to track engineering performance without adding overhead.
- **Real-World Case Studies:** Anonymized data on how CRAFTER solved specific organizational problems.
- **Tooling/Automation:** Simple scripts or templates that automate the [Playbook](./Playbook.md).

---

## ❌ What We Reject

To maintain high signal-to-noise ratio, we do not accept:
- **Vague Philosophies:** Ideas that sound good but lack a "How-to" component.
- **Over-Engineering:** Adding complexity to solve a problem that doesn't exist at scale.
- **Unnecessary Abstractions:** Theoretical models that haven't been tested in the "trenches."
- **Feature Creep:** Adding modules that distract from the core mission of **High Performance.**

---

## 🛠 The Process (The CRAFTER Way)

1. **Identify the Friction:** Open an **Issue** describing the specific problem or inefficiency you've found.
2. **Fork & Branch:** Work on your solution in a dedicated branch.
3. **Self-Assessment:** Run your own contribution through the [CRAFTER Principles](./CRAFTER.md) before submitting.
4. **Submit PR:** Provide a clear explanation of:
   - What problem is being solved?
   - How does this improve engineering execution?
   - Is there a template or checklist included?

---

## 📜 The Golden Rule

> **If it does not improve execution — it does not belong here.**
