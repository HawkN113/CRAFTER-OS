# 🧪 Self-Assessment Protocol (CRAFTER)

> **Goal:** Objective measurement of engineering maturity. Use this during your **Weekly Review** to identify system bottlenecks.

---

## 📊 Scoring System
Rate each principle from **0 to 5**:
- **0:** Total failure/chaos.
- **5:** Perfect execution/mastery.

---

## 🔤 The C.R.A.F.T.E.R Audit

### 🔵 C — Cognitive Clarity
- [ ] **Understanding:** Did I start every task with a 100% clear "Definition of Done"?
- [ ] **Simplicity:** Can I explain my current architecture to a non-technical peer?
- [ ] **Alignment:** Do I know *why* I am building this feature for the business?
*Score: [ ] / 5*

### ⚪ R — Reputation
- [ ] **Reliability:** Did I meet 90%+ of my internal and external commitments?
- [ ] **Transparency:** Did I communicate delays *before* the deadline, not after?
- [ ] **Quality:** Is my code something I am proud to have my name on?
*Score: [ ] / 5*

### 🟢 A — Adaptation
- [ ] **Resilience:** How did I react when requirements changed mid-sprint?
- [ ] **Learning:** Did I update my [Playbook](./Playbook.md) after a mistake?
- [ ] **Pivoting:** Did I stop dead-end approaches quickly, or did I "sink cost"?
*Score: [ ] / 5*

### 🔴 F — Focus
- [ ] **Deep Work:** Did I achieve at least 2–4 hours of "No-Noise" execution daily?
- [ ] **Boundary Setting:** Did I say "No" to non-essential pings/meetings?
- [ ] **Flow State:** How many times did I reach a state of intense concentration?
*Score: [ ] / 5*

### 🟡 T — Technical Mastery
- [ ] **Skill Delta:** Did I learn one new pattern, tool, or optimization this week?
- [ ] **Root Cause:** Did I fix the *source* of bugs, or just the symptoms?
- [ ] **Mentorship:** Did I contribute to the collective knowledge (Docs/PR reviews)?
*Score: [ ] / 5*

### 🟠 E — Execution
- [ ] **Completion:** Did I finish what I started, or is my backlog growing?
- [ ] **Velocity:** Is my "Time-to-PR" decreasing or stabilizing?
- [ ] **Decisiveness:** Did I avoid "Analysis Paralysis" and move to action?
*Score: [ ] / 5*

### 🟣 R — Responsibility
- [ ] **Ownership:** Did I treat the codebase like I own the company?
- [ ] **Proactivity:** Did I fix a problem I saw, even if it wasn't "my task"?
- [ ] **Accountability:** Did I take full credit for my failures without blaming "the legacy code"?
*Score: [ ] / 5*

---

## 📉 Diagnostic Rules

| Total Score | Status | Action Required |
|:---:|:---|:---|
| **30–35** | 🚀 **High Performer** | Increase complexity; mentor others. |
| **20–29** | ⚖️ **Stable** | Identify the one "weakest link" and fix it. |
| **< 20** | ⚠️ **System Crisis** | Stop new tasks. Restore **Clarity** and **Focus** immediately. |

> **Critical Note:** If more than two categories score below **3**, the system is in "Fragile" state. Do not ignore the signal. **Recalibrate the Playbook.**
