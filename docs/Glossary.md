# 📖 Glossary: CRAFTER OS

| Term | System Definition | Principle |
|:---|:---|:---|
| **ADR (Architecture Decision Record)** | A document that captures a significant technical decision, its context, and its consequences. The "Black Box" of project evolution. | **C / A** |
| **Cognitive Clarity** | A state of 100% understanding of a task's logic and "Definition of Done" before writing code. | **C** |
| **Deep Work** | A 2-4 hour block of high-intensity focus with zero notifications or context switching. | **F** |
| **Signal-to-Noise** | The ratio of high-impact engineering work (code, architecture) to low-value friction (vague syncs, chat). | **C** |
| **Antifragility** | The property of a system or engineer to get stronger/better as a result of stressors and failures. | **A** |
| **Definition of Done** | A strict, non-negotiable checklist that must be met for a task to be considered complete (Tests, Docs, PR). | **E** |
| **Reputation Score** | A measure of reliability based on the delta between commitments made and results delivered. | **R** |
| **Broken Window** | A small bug, poorly written comment, or tech debt that degrades system quality. Must be fixed immediately. | **R** |
| **Analysis Paralysis** | A failure state where execution stops due to over-thinking. Solved via the "20-Min Mapping Rule". | **C** |
